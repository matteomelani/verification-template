# Project Invariants

The canonical list of things this system must *never* do. Each invariant is enforced by one or more contracts in `contracts/`.

This file is a fast lookup: scan the headlines, find the relevant invariant, click through to the contracts that enforce it.

Invariants are added carefully and removed almost never. They are the "constitutional law" of the codebase.

---

## Format

```
## INV-NNN: <one-sentence statement of the invariant>
- Enforced by: contracts/NNNN-name.md, contracts/NNNN-name.md
- Why it exists: <one paragraph>
- Status: active | deprecated
```
